# 探索8: 追加検証例

## 目的
IsGoldbach の検証例をより大きな偶数に拡張し、norm_num による自動証明の限界を探る。

## 追加した検証例

| n | 分解 | 備考 |
|---|------|------|
| 200 | 3 + 197 | 197 は素数 |
| 500 | 13 + 487 | 487 は素数。497 は素数でない (497 = 7 × 71) |
| 1000 | 3 + 997 | 997 は素数 |

## 素数の確認
- 197: 素数（14² = 196 < 197 なので 13 以下の素数で割れないことを確認）
- 487: 素数（22² = 484 < 487 なので 19 以下の素数で割れないことを確認）
- 497: 合成数 (497 = 7 × 71) → 500 = 3 + 497 は使えない
- 493: 合成数 (493 = 17 × 29) → 500 = 7 + 493 も使えない
- 997: 素数（31² = 961 < 997 なので 31 以下の素数で割れないことを確認）

## 成果
- 200, 500, 1000 の3つの検証例を追加
- norm_num で 3桁の素数判定も自動的に処理可能であることを確認
- 累計検証例: 4, 6, 8, 10, 12, 20, 100, 200, 500, 1000 の10個
