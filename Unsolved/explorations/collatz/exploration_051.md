# Exploration 051: Delay Records の系列と成長率分析

## 概要
Collatz delay record（stopping time の最大記録を更新する数 n の列）について、n = 1..10^7 の範囲で系統的に分析した。

## スクリプト
`scripts/collatz_delay_records.py`

## 結果

### 1. Delay Records 一覧
10^7 までに **54個** の delay record が存在。

| k | n | stopping time | 主な特徴 |
|---|---|---|---|
| 0 | 1 | 0 | 2^1-1 |
| 4 | 7 | 16 | 2^3-1 |
| 8 | 27 | 111 | 大ジャンプ（st比4.83倍）|
| 30 | 26623 | 307 | trailing 1s = 11 |
| 43 | 837799 | 524 | 有名な delay record |
| 53 | 8400511 | 685 | 最大（10^7以内）|

### 2. 成長率分析

**n の連続比 r_{k+1}/r_k:**
- 平均: 1.3625
- 中央値: 1.3330
- 範囲: [1.0414, 2.0000]

**指数フィット:**
```
r_k ≈ 3.205 × 1.3372^k
```
- alpha = 1.3372 は 4/3 = 1.3333... に極めて近い
- これはコラッツ写像の「3n+1 で 3/2 倍、/2 で 1/2 倍」の平均的拡大率 3/2 × 1/2 × 2 ≈ 4/3 と整合
- 2^(1/1) = 2.0 や 3^(1/2) = 1.732 とは一致しない

**stopping time の比 s_{k+1}/s_k:**
- 平均: 1.2548（外れ値の影響大）
- 中央値: 1.0316（通常は微増）
- 大ジャンプは稀（n=3→7で7倍、n=27で4.83倍）

### 3. 2進表現の特徴

- **1-bit密度**: 平均約0.55-0.65、特に高い密度（0.8以上）を持つものが散在
- **trailing 1s**: delay recordは末尾に連続する1を持つ傾向
  - n=26623: trailing 1s = 11（15ビット中13が1）
  - n=3711: trailing 1s = 7
  - n=156159: trailing 1s = 9
- trailing 1sが多い → 最初の数ステップで急激に値が増大

### 4. メルセンヌ数との関係

2^k-1 形の delay record: **3/54 (5.6%)** のみ
- 2^1-1 = 1, 2^2-1 = 3, 2^3-1 = 7

小さい値でのみメルセンヌ数が delay record。大きくなると全ビット1の数は stopping time が十分長くならない。ただし、部分的にビットが密（trailing 1s が多い）パターンは delay record に頻出。

### 5. Mod パターン

**mod 2**: 奇数が **93%** を占める（偶数の delay record は 2n が自明に作るもの）

**mod 4**: 3 mod 4 が **67%** — 奇数の中でも 3 mod 4 が圧倒的に多い
- 3 mod 4 の数は 2進表現の下位2ビットが11 → 最初のステップで 3n+1 = 3(4k+3)+1 = 12k+10 → /2 = 6k+5 (奇数) → すぐにまた3n+1が適用され、急激に成長

**mod 8**: 7 mod 8 が **50%** — 下位3ビットが111
- 同様の理由で3ステップ連続で奇数を維持し、急成長する

**mod 3**: 0 mod 3 が **57%** — 3の倍数が過半数
- これは delay record が 3k-1 の形（...111 パターン）を好むことの反映

### 6. Glide Records

10^6 までに **38個** の glide record（最大値に到達するまでのステップ数が最大の n）。

delay record と glide record の重複: **18個** (10^6以内)

主要な非重複 glide record:
- n=270271 (glide=211), n=360361 (glide=214) — 7の倍数の系列
- n=919791 (glide=236) — 最大の glide record

### 7. Stopping Time のスケーリング

stopping time s(n) と log2(n) の比:
- 全体平均: **19.36**
- 後半20個の平均: **25.34**
- 比は増加傾向にある → s(n) は O(log n) より速く成長する可能性

理論的にはランダム写像モデルで s(n) ~ C * (log n)^2 が予測されるが、10^7 の範囲では st/log2(n) の増加が st/(log2(n))^2 の減少に転じるかは判断困難。

## 主要な発見

1. **成長定数 α ≈ 4/3**: delay record の指数的成長率がコラッツ写像の平均拡大率 4/3 と一致。これは確率論的モデルの予測と整合する。

2. **奇数・3 mod 4 バイアス**: delay record の 93% が奇数、67% が 3 mod 4。2進表現で末尾に1が連続する数が delay record になりやすい。

3. **Trailing 1s の重要性**: 2進表現の末尾に連続する1を多く持つ数は、初期段階で急激に値が増大し、長い軌道を生みやすい。

4. **Stopping time の超対数的成長**: st/log2(n) 比が増加傾向を示し、delay record の stopping time は O(log n) よりやや速く成長する。

5. **Glide record との部分的重複**: 初期の delay record はほぼ全て glide record でもあるが、大きい値では分岐する。

## 形式化への示唆
- delay record の成長率 α ≈ 4/3 は、Lean で `∀ ε > 0, ∃ C, ∀ k, delay_record k ≤ C * (4/3 + ε)^k` として定式化可能
- mod 4 での分布偏りは、`delay_record k % 4 = 3` が「ほぼ全ての」k で成立することの形式化に繋がる
