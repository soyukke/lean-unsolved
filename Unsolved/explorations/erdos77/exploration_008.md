# 探索8: 2色塗り分けの基本的な性質

## 目標
- 2色塗り分けにおける辺の色の場合分け補題を形式化
- C5着色の全辺を decide で網羅的に検証

## 結果

### R8a: two_coloring_color_cases
- **定理**: 任意の辺 (i,j) に対し `col.color i j = true ∨ col.color i j = false`
- **証明**: `Bool.eq_true_or_eq_false` を直接適用
- **意義**: ラムゼー性質の証明で辺の色で場合分けする際の基盤補題

### R8b: two_coloring_self
- **定理**: `col.color i i = col.color i i`
- **証明**: `rfl`
- **意義**: 自己ループ（対角要素）は定義上問題にならないことの確認

### R8c: C5着色の全辺の色の検証
decide ベースで C5 の各辺の色を網羅的に検証:

c5Adjacent の定義: `d = (i + 5 - j) % 5` が 1 または 4 なら隣接(true)。

**隣接辺 (色true)**: (0,1), (1,2), (2,3), (3,4), (0,4) — C5 の5辺
**非隣接辺 (色false)**: (0,2), (0,3), (1,3), (1,4), (2,4) — C5 の補グラフの5辺

C5 の隣接構造:
```
0 -- 1 -- 2 -- 3 -- 4 -- 0
```

具体的な計算:
- (0,1): d = (0+5-1)%5 = 4 → true
- (1,2): d = (1+5-2)%5 = 4 → true
- (2,3): d = (2+5-3)%5 = 4 → true
- (3,4): d = (3+5-4)%5 = 4 → true
- (0,4): d = (0+5-4)%5 = 1 → true
- (0,2): d = (0+5-2)%5 = 3 → false
- (0,3): d = (0+5-3)%5 = 2 → false
- (1,3): d = (1+5-3)%5 = 3 → false
- (1,4): d = (1+5-4)%5 = 2 → false
- (2,4): d = (2+5-4)%5 = 3 → false

全10辺を decide で検証し、C5 構造が正しく定義されていることを確認した。

## 技術的メモ
- `Bool.eq_true_or_eq_false` は Lean 4 の標準ライブラリで提供
- C5 の具体的な辺は Fin 5 上で decide が効率的に動作する
- ビルド環境の制約により、実際のコンパイル検証は次回に持ち越し
