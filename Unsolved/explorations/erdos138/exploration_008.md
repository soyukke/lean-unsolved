# 探索8: N=4,5,6,7 での 3-AP 回避検証と 4-AP の検証

## 目標
- N=4,5,6,7 での 3-AP (k=3) 回避可能な塗り分けの具体例を rfl で検証
- 4-AP (k=4) に関する基本的な回避可能性の検証

## 結果

### W8a: N=4,5,6,7 での 3-AP 回避
全て `hasMonoAPList ... 3 = false` を `rfl` で証明:
- **N=4**: RBRB `[F,T,F,T]` — 3-AP 回避可能
- **N=5**: RBRBB `[F,T,F,T,T]` — 3-AP 回避可能
- **N=6**: RBBRRB `[F,T,T,F,F,T]` — 3-AP 回避可能
- **N=7**: RBBRRBB `[F,T,T,F,F,T,T]` — 3-AP 回避可能

N=8 での回避可能（既に探索3で検証済み: RRBBRRBB 等）と合わせて、
N ≤ 8 での 3-AP 回避可能性を系統的に網羅した。

### W8b: 4-AP (k=4) の回避可能性
- **N=3**: RBR — 3点に4項APは入らない（自明）
- **N=5**: RBRBR — 4-AP 回避可能
- **N=7**: RBRBRBB — 4-AP 回避可能
- **N=10**: RRRBRRBRRR `[F,F,F,T,F,F,T,F,F,F]` — 4-AP 回避可能

W(4) = 35 なので、N ≤ 34 まで回避可能な塗り分けが存在するはず。
N=10 での具体例の構成に成功。

- 4-AP の回避塗り分けは 3-AP より見つけやすい（制約が緩い）
