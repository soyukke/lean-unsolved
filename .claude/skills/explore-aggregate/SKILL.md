# 探索結果の集約

## 対象問題: $ARGUMENTS

## 手順

### 1. 集約対象の特定

1. `Unsolved/explorations/queue.json` を読む
2. `status: "done"` かつ前回集約後のアイテムを収集
3. 各アイテムの `result_file` から探索結果ファイルを読む

### 2. 探索結果の横断分析

以下の観点で直近の探索結果を横断的に分析する:

#### 2a. 発見の整理
- **確定した事実**: 証明済み or 数値的に確認済みの結果
- **有望な仮説**: 未証明だが傍証がある主張
- **否定された方向**: 明確に失敗した or 非生産的だったアプローチ

#### 2b. パターンの抽出
- 複数の探索に共通して現れる構造はあるか？
- 「Aの結果」と「Bの結果」を組み合わせると新しい知見が得られないか？
- 想定外の結果はあったか？（セレンディピティの検出）

#### 2c. カバレッジ分析
以下の10カテゴリのうち、まだ手薄なものを特定:
計算実験 / 代数的構造 / 解析的手法 / 組合せ論 / 確率的 / 位相幾何 / 論理 / 補題拡張 / 変種 / 逆問題

### 3. 知識ベースの更新

#### 3a. INDEX.md の整備
`Unsolved/explorations/$ARGUMENTS/INDEX.md` を確認し、未記載の探索があれば追加。

#### 3b. サマリーファイルの更新
`Unsolved/explorations/$ARGUMENTS/SUMMARY.md` を作成 or 更新:

```markdown
# [問題名] 探索サマリー

## 最終更新: [日付]

## 確定した事実
- [証明済み結果1]
- [証明済み結果2]

## 有望な方向
1. [方向A]: [根拠]
2. [方向B]: [根拠]

## 行き止まり（再探索不要）
- [方向X]: [失敗理由]

## 形式化状況
- sorry なし: N 件
- sorry あり: M 件
- 未形式化: K 件

## カバレッジ
| カテゴリ | 探索数 | 最新 | 成果 |
|---------|--------|------|------|
| 計算実験 | ... | ... | ... |
```

### 4. 次のアクション判定

以下の条件で判定する:

| 条件 | アクション |
|------|-----------|
| キューに `queued` が3件以上残っている | → `/explore-cycle` を継続 |
| キューが残り少ない（0-2件） | → `/brainstorm $ARGUMENTS` を推奨 |
| 大発見/ブレイクスルーがあった | → 形式化を優先。Lean証明の作成を推奨 |
| 5回以上連続で「成果なし」 | → 問題の切り替え or 根本的な再アプローチを推奨 |

### 5. queue.json の更新

```json
{
  "last_aggregate": "2026-03-27",
  "aggregate_count": N
}
```

### 6. 出力

以下をユーザに報告:
- 集約した探索件数
- 主要な発見（あれば）
- カバレッジの偏り
- 次のアクション推奨
